# 修复注册和登录API返回HTML的错误

## 为什么

当前，在开发环境中，前端对 `/api/register` 和 `/api/login` 的请求错误地收到了一个HTML页面（`index.html`），而不是预期的JSON响应。这是因为Vite开发服务器处理了这些本应被代理到后端的API请求。这导致客户端出现JSON解析错误，使得注册和登录功能无法使用。

## 变更内容

- **修改 `server.js`**:
  - 添加静态文件服务，用于在生产环境中为前端应用提供服务。
  - 添加一个通配符路由，将所有非API请求重定向到 `index.html`，以支持客户端路由。

- **修改 `package.json`**:
  - 修改 `dev` 脚本，确保前端和后端服务可以并行启动。
  - 添加一个 `start` 脚本，用于在生产环境中启动服务器。

## 影响

- **受影响的规范**:
  - 用户认证
- **受影响的代码**:
  - `server.js`: 后端服务器入口文件。
  - `package.json`: 项目脚本和依赖项配置文件。

## 修改的需求

### 需求: API响应格式

系统应确保所有对 `/api/*` 端点的请求都返回有效的JSON响应。

#### 场景: 成功的API调用
- **当** 客户端使用有效数据向 `/api/register` 发送POST请求时
- **那么** 服务器应返回一个JSON对象（例如 `{"message": "用户创建成功"}`）和201状态码。

#### 场景: API调用到不存在的端点
- **当** 客户端向一个未定义的 `/api` 端点发送请求时
- **那么** 服务器应返回404状态码和JSON格式的错误消息，而不是HTML页面。
